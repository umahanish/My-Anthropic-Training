# Working With Claude — Student Lab Guide
### Session 2: From First Chat to Connected Workflows

This is your copy — labs only, no lecture. Every lab is something you actually do, live, in Claude. Work through the **Core** labs in each module first; if you finish early, move to **Stretch** labs rather than waiting on the group.

**Legend:** 🟢 Core (everyone does this) · 🔵 Stretch (bonus — dig in if you have time)

**Before you start:** A few labs today need a paid plan (Pro, Max, Team, or Enterprise) and a couple of features are still in beta or rolling out. If a tab or feature mentioned below isn't visible to you, that's expected, not a mistake on your end — flag it to your facilitator and move to the next lab rather than getting stuck.

---

## Module 1 — What is Claude? First Conversations & Getting Better Results

### 🟢 Lab 1.1 — Your First Real Conversation *(10 min)*
1. Open Claude and ask it something real you actually need today — not a test question.
2. Look at the response. Instead of asking a brand-new question, **iterate**: tell Claude specifically what to change, add, or fix.
3. Do this once more. By your third message, the response should be noticeably closer to something you'd actually use.

### 🟢 Lab 1.2 — Upload & Ask *(8 min)*
1. Upload one real file — a document, spreadsheet, or image.
2. Ask Claude a specific question about it (not "summarize this," something you actually need to know).
3. Now ask the same *kind* of question without the file, from memory/description alone. Compare the two answers — notice the gap in specificity and accuracy.

### 🔵 Lab 1.3 — Build on a Past Chat *(7 min)*
1. Find an older conversation you've had with Claude (or one from earlier today).
2. In a *new* conversation, ask Claude to recall or build on that earlier chat using its chat search/memory feature.
3. Notice whether it retrieves the right context — and if not, try being more specific about which past conversation you mean.

## Module 2 — Claude Desktop: Chat, Cowork, Code

### 🟢 Lab 2.1 — Route the Task *(12 min, pairs)*
For each task below, decide: **Chat**, **Cowork**, or **Code** — and write one sentence justifying it.
1. "Summarize this one email thread for me."
2. "Read every contract in this folder and flag which ones expire in the next 90 days."
3. "Fix the failing test in this repository."
4. "Set up a task that emails me a summary of yesterday's Slack activity every morning."

Compare answers with your partner — where did you disagree, and why?

### 🟢 Lab 2.2 — Same Task, Two Modes *(10 min)*
1. Pick one simple, real task you could ask Claude to do.
2. Run it once in **Chat**.
3. Run the *same* task in **Cowork** (if you have access).
4. Compare: what did Cowork do differently — more sources, a saved file, a plan you could review? Was that extra depth actually useful for this task, or overkill?

### 🔵 Lab 2.3 — Peek at Code *(10 min, if you have Code tab access)*
1. Open the Code tab and select a small local project (or a sandbox repo if you don't have one handy).
2. In **Ask** mode, request one small, low-risk change.
3. Review the visual diff Claude proposes *before* approving anything — this is the same habit as reviewing a colleague's pull request.

## Module 3 — Introduction to Projects, Artifacts & Skills

### 🟢 Lab 3.1 — Build Your Project *(12 min)*
1. Create a new Project around something real you're working on.
2. Add custom instructions — at least one sentence on tone or format you want Claude to default to.
3. Upload or paste in 1–2 real documents relevant to that work.
4. Ask a question inside the Project that requires it to use what you just gave it. Does the answer actually reflect your documents and instructions, or does it feel generic?

### 🟢 Lab 3.2 — Generate & Iterate an Artifact *(10 min)*
1. Ask Claude for something substantial — code, a formatted document, a diagram, a small interactive tool.
2. Once it appears in the Artifact window, ask for one specific change.
3. Confirm it updated the *same* Artifact in place rather than starting over from scratch.

### 🔵 Lab 3.3 — Trigger a Skill on Purpose *(10 min)*
1. Ask Claude to produce something as a real file — for example, "turn this into a formatted Word document" or "build this into an Excel sheet with formulas."
2. Open the resulting file and check the formatting quality — headings, structure, formulas actually working (not just static numbers).
3. That polish is a Skill firing automatically in the background. Notice what triggered it — what phrase or request type set it off?

## Module 4 — Connecting Tools: Enterprise Search & Research Mode

### 🟢 Lab 4.1 — Ask Your Org *(10 min)*
1. If Enterprise Search is set up for your organization, open the "Ask [Your Org]" project.
2. Ask a real question you'd normally have to hunt for across multiple tools (email, chat, docs).
3. Check: did the answer pull from more than one source? Can you trace each claim back to where it came from?

*(Not set up yet? Watch your facilitator demo this live instead, and move to Lab 4.2.)*

### 🟢 Lab 4.2 — Turn On Research *(12 min)*
1. Confirm web search is turned on — Research needs it to function.
2. Turn on Research mode.
3. Ask a genuinely open-ended question relevant to your work — something needing real digging, not a single-fact lookup.
4. While it runs, watch what it's checking along the way. When it's done, click into at least **one** citation and confirm it actually says what Claude claims it says.

### 🔵 Lab 4.3 — Steer Research *(8 min)*
1. Run a Research query where you suspect Claude might miss an obvious source (e.g., a niche or internal one).
2. If it doesn't check that source on its own, steer it explicitly: *"Claude, please pull relevant context from [specific source]."*
3. Notice how much the answer improves with one targeted nudge versus a fully open-ended ask.

## Module 5 — Claude Cowork: Standing Context, Plugins & Team Sharing

### 🟢 Lab 5.1 — Set Global Instructions *(10 min)*
1. Go to Settings > Cowork > Global instructions.
2. Write 2–3 sentences that should apply to *everything* Cowork does for you — your role, tone, or a standing preference.
3. Run a quick task in Cowork afterward and see whether the output reflects what you just set.

### 🟢 Lab 5.2 — Create a Cowork Project (and Test Its Memory) *(12 min)*
1. Create a Cowork Project pointed at a real folder you work in. Add project-specific instructions on top of your global ones.
2. Run one real task inside that Project.
3. A few minutes later, run a second, *related* task in the same Project. Does Claude reference anything from the first task without you re-explaining it? That's memory working.

### 🔵 Lab 5.3 — Install & Try a Plugin *(8 min)*
1. If plugins are available to your org, browse what's on offer in the Cowork interface.
2. Install one relevant to your role.
3. Run one task with it active, and one without. Compare — what did the plugin actually add?

## Module 6 — Claude in Chrome & Microsoft 365 Integration

### 🟢 Lab 6.1 — Supervised Browser Task *(12 min)*
1. Open Claude in Chrome (confirm with your facilitator it's approved for use first).
2. Pick a low-stakes, real task — research something across 2–3 sites, or extract information from a page you'd normally copy-paste manually.
3. Before approving any action Claude proposes, actually read what it's about to do. Don't reflexively click approve.
4. Once it works, save the prompt as a shortcut for next time.

### 🟢 Lab 6.2 — One Office Edit *(10 min)*
1. Open Word, Excel, or PowerPoint (whichever you use most) and activate the Claude add-in.
2. Give it one real, small edit request — reformat a section, build a formula, restructure a slide to match your template.
3. Check the result properly: in Word, actually read the tracked change before accepting it; in Excel, check the formula itself, not just the displayed number.

### 🔵 Lab 6.3 — Cross-App Handoff *(8 min)*
1. Open an Excel workbook with some real data, and a PowerPoint file, side by side.
2. Ask Claude (from either app's sidebar) to pull specific numbers from the Excel file directly into a slide in the PowerPoint file.
3. Confirm it happened without you copying or pasting anything manually — that's the cross-app context feature at work.
